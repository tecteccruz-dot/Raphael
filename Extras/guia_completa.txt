Arquitectura GeneralEl juego es una aplicación web de una sola página (SPA) que corre completamente en el navegador. No se necesita backend propio. La IA externa funciona como Dungeon Master a través de una API configurada por el jugador.1.1 Las Tres CapasTodo el sistema se divide en tres capas que se comunican entre sí:CapaTecnologiaResponsabilidadFrontendReact / HTML5Interfaz visual, formularios, pantallas del juegoMotor de EstadoJavaScript + localStoragePersonaje, inventario, mundo, historial de turnosIA / DMAzure Copilot / ChatGPT / LM StudioNarración, decisiones, comandos de estado1.2 Flujo de ComunicaciónEl ciclo de cada turno funciona así:El jugador escribe su acción en el chat de la pantalla de juego.El motor de estado ensambla el contexto completo: prompt maestro + estado actual + historial reciente + acción del jugador.Se envía ese paquete a la API de la IA.La IA responde con narración + comandos de estado (en formato estructurado).El frontend parsea la respuesta, muestra la narración y ejecuta los comandos.El estado se actualiza y se guarda en localStorage.1.3 Orden de Construcción RecomendadoConstruye en este orden para tener algo jugable lo antes posible:Configuración de API (sin esto nada funciona)Pantalla de juego básica: solo el chat con la IA, sin estado aúnPrompt maestro base con toda la lore predeterminadaMotor de estado: inyectar el estado en cada promptParser de comandos: la IA puede modificar el estadoAsistente de creación de personaje: paso a pasoLos tres inicios de partida: sus prompts de arranqueGalería de personajes: guardar y cargarFondos, arte e ilustracionesSistema de guardado con slots y compresión de historial2. Asistente de Creación de PersonajeEs un wizard de pasos guiados, no un formulario plano. Cada paso depende del anterior. El jugador puede retroceder en cualquier momento.Paso 1 — Nombre y SexoCampo de texto libre para el nombre del personajeSelector de sexo: Masculino, Femenino, Otro (sin efecto en mecánicas, solo en narración)Paso 2 — Raza y SubrazaTarjetas visuales con ilustración, nombre y descripción breve de cada razaAl elegir una raza, se despliegan sus subrazas disponibles en el mismo pasoLa raza otorga bonificaciones de estadística y rasgos pasivos automáticamentePaso 3 — Clase y SubclaseTarjetas de clase con icono representativo, descripción de rol y estadística principalLa subclase se muestra desde el principio para que el jugador vea hacia dónde vaLa clase define las proficiencias de armas, armaduras y habilidades disponiblesPaso 4 — TrasfondoLos trasfondos son los de DnD 5E. La estrategia es:Mantener los últimos 12-15 turnos completos en el contextoCada 10 turnos, pedir a la IA que genere un 'resumen comprimido' de lo que pasó antesEse resumen reemplaza los turnos viejos en el contexto, ocupando mucho menos espacioEl resumen comprimido se inyecta al inicio del historial como un bloque especialEjemplo de resumen comprimidoRESUMEN: El jugador llegó a Tumbaflor, habló con el tabernero Gordo Wren yaceptó la misión de entregar una carta al herrero. En el camino fue atacadopor dos bandidos y los derrotó usando el hechizo Manos Ardientes. Perdió5 HP y gastó una ranura de hechizo de nivel 1. Ahora tiene 3 HP y buscadescanso. El herrero aún no ha sido encontrado.5. El Prompt MaestroEs el documento más importante del juego. Convierte a la IA genérica en un Dungeon Master consistente con identidad propia. El frontend divide la respuesta en sus partes:NARRACION:[Texto narrativo para el jugador, 2-4 párrafos, segunda persona, presente]COMANDOS:[DAMAGE player 5][REMOVE_ITEM pocion_curacion]OPCIONES:1. Atacar al goblin con tu espada2. Intentar negociar con el goblin herido3. Explorar el cuerpo del jefe de la mazmorraLa NARRACIÓN va al panel de chat. El jugador la lee.Los COMANDOS se ejecutan silenciosamente. El jugador no los ve (o aparecen en un log técnico pequeño).Las OPCIONES aparecen como botones de sugerencia rápida que el jugador puede ignorar o clickear.6. El jugador puede exportar su partida como archivo JSON para hacerle respaldo.8.1 Estructura de localStorageClaveContenidorpg_configURL del endpoint, API Key, modelo de IA elegidorpg_prompt_maestroEl texto completo del prompt maestro (versión del jugador)rpg_galeriaArray de personajes guardados en JSONrpg_personaje_activoID del personaje actualmente seleccionadorpg_partida_activaEstado completo del mundo + historial de turnos activorpg_slotsArray de hasta 5 slots de guardado manual con fecha y captura de estado8.2 Guardado AutomáticoEl estado se guarda automáticamente al final de cada turnoSi el jugador cierra el navegador, al volver encuentra su partida exactamente donde la dejóUn indicador visual discreto muestra 'Guardado' por 2 segundos tras cada autosave8.3 Slots de Guardado Manual5 slots disponibles, el jugador puede guardar en cualquier momento desde el menúCada slot muestra: fecha, hora, ubicación, HP del personaje y nombreCargar un slot regresa al estado exacto de ese momentoLos slots se pueden sobrescribir o borrar8.4 Exportar / ImportarEl jugador puede exportar su partida completa como archivo .jsonEse archivo se puede importar en otro dispositivo o como respaldoTambién se puede exportar solo el personaje para compartir con otros jugadores9. Se llama cada vez que el jugador envía una acción:async function procesarTurno(accionJugador) {  // 1. Ensamblar el contexto completo  const contexto = {    system: promptMaestro + promptInicio,    messages: [      ...historialComprimido,      ...ultimosTurnos,      { role: 'user', content: estadoActual + accionJugador }    ]  };  // 2. Llamar a la IA  const respuesta = await llamarAPI(contexto);  // 3. Parsear la respuesta  const { narracion, comandos, opciones } = parsearRespuesta(respuesta);  // 4. Ejecutar comandos  comandos.forEach(cmd => ejecutarComando(cmd));  // 5. Actualizar UI  agregarAlChat(narracion);  mostrarOpciones(opciones);  // 6. Guardar estado  guardarEstado();}9.4 Manejo de ErroresSi la API falla: mostrar mensaje 'El oráculo no responde...' y permitir reintentarSi la respuesta no tiene el formato correcto: mostrar la respuesta completa en el chat sin parsearSi localStorage está lleno: advertir al jugador y ofrecer limpiar historial antiguoSi la clave API es incorrecta: llevar al jugador a la pantalla de configuración10.
